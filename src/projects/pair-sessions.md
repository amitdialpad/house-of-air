## Why I built it

Designers at Dialpad keep asking how I work with Claude. Sending links and Loom videos doesn't stick. The shift in process is bigger than "use this tool". It's a different rhythm. You learn it by doing it once with someone.

## What it is

Three structured sessions, like pair programming for designers. Session 1 is "my turn". The facilitator walks through a real, in-flight task while you watch and interrupt anytime. Session 2 flips: you drive. Session 3 is on shipping, design judgment, and accessibility.

The sessions cover the full rhythm. PRD analysis with Claude, internal signals (Amplitude, Jira, code context), shaping requirements, breadboarding, building in vertical slices, PR prep. Not slides. Not a prepared demo. Real work.

## How it lives

A public site (same repo that hosts Beacon Brief). Anyone on the team can read the sessions on their own, but the value is doing them with a partner. No certificates, no homework. Just a structured way to pick up the new rhythm without faking your way through it.

## Day to day

Run a session when a designer asks. Update the content when the toolkit changes.
